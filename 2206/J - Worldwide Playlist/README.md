<div class="header"><div class="title">J. Worldwide Playlist</div><div class="time-limit"><div class="property-title">time limit per test</div>2 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>1024 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p>You are planning a trip around the world. For this trip, you have installed a music app containing $$$n$$$ songs, numbered from $$$1$$$ to $$$n$$$.</p><p>Initially, the app generates a playlist for these $$$n$$$ songs in the form of a permutation of $$$1, 2, \ldots, n$$$, denoted by $$$a_1, a_2, \ldots, a_n$$$. It plays the $$$n$$$ songs in the order $$$a_1, a_2, \ldots, a_n$$$: song $$$a_1$$$ plays first, song $$$a_2$$$ plays second, and so on. The playlist is <span class="tex-font-style-underline">infinite</span>: each time after song $$$a_n$$$ plays, it starts all over from song $$$a_1$$$.</p><p>The next song starts playing automatically when the current song finishes. Before a song finishes, you may instead press a <span class="tex-font-style-underline">skip</span> button to immediately jump to the next song in the playlist.</p><p>You have a desired permutation of the $$$n$$$ songs, denoted by $$$b_1, b_2, \ldots, b_n$$$. This means that you want to listen to $$$n$$$ songs <span class="tex-font-style-underline">in full</span> in the order of $$$b_1, b_2, \ldots, b_n$$$ by strategically pressing the skip button zero or more times. In other words, you want the first song you listen to in full (not skipped) to be song $$$b_1$$$, the second to be song $$$b_2$$$, and so on, until the $$$n$$$-th is song $$$b_n$$$. After listening to these $$$n$$$ songs in full, you stop listening.</p><p>You use this playlist for $$$d$$$ days. Between each pair of consecutive days, you update the permutations using three integers $$$c$$$, $$$x$$$, and $$$y$$$ as follows: </p><ul> <li> If $$$c = 1$$$, then you swap $$$a_x$$$ and $$$a_y$$$. </li><li> If $$$c = 2$$$, then you swap $$$b_x$$$ and $$$b_y$$$. </li></ul><p>Note that the effect of each update persists to subsequent days.</p><p>For each day, assuming you start listening from song $$$a_1$$$, determine the minimum number of times you need to press the skip button so that the songs you listen to in full are $$$b_1, b_2, \ldots, b_n$$$ in this order.</p></div><div class="input-specification"><div class="section-title">Input</div><p>The first line of input contains two integers $$$n$$$ and $$$d$$$ ($$$2 \le n \le 200\,000$$$; $$$2 \le d \le 200\,000$$$).</p><p>The second line contains $$$n$$$ integers representing the initial values of $$$a_1, a_2, \ldots, a_n$$$ ($$$1 \le a_i \le n$$$; $$$a_i \neq a_j$$$ for all $$$i \neq j$$$).</p><p>The third line contains $$$n$$$ integers representing the initial values of $$$b_1, b_2, \ldots, b_n$$$ ($$$1 \le b_i \le n$$$; $$$b_i \neq b_j$$$ for all $$$i \neq j$$$).</p><p>The $$$k$$$-th of the next $$$d-1$$$ lines contains three integers $$$c$$$, $$$x$$$, and $$$y$$$ ($$$c \in \{1, 2\}$$$; $$$1 \le x  \lt  y \le n$$$), representing the update between the $$$k$$$-th and $$$(k+1)$$$-th days.</p></div><div class="output-specification"><div class="section-title">Output</div><p>Output $$$d$$$ lines, where the $$$k$$$-th line should contain the minimum number of skips for the $$$k$$$-th day.</p></div><div class="sample-tests"><div class="section-title">Examples</div><div class="sample-test"><div class="input"><div class="title">Input<div title="Copy" data-clipboard-target="#id00574779386420181" id="id0007618032922860318" class="input-output-copier">Copy</div></div><pre id="id00574779386420181">4 3
