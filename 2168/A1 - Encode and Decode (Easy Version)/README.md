Encode and Decode (Easy Version)</div><div class="time-limit"><div class="property-title">time limit per test</div>2 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>512 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p><span class="tex-font-style-bf">This is the easy version of the problem. The difference between the versions is that in this version, $$$a_i \leq 26$$$.</span></p><p><span class="tex-font-style-it">This is a run-twice (communication) problem.</span> In these problems, your program will be run twice. All variables stored in the memory will be lost between runs, but information given to you in the first run may be important in trying to solve the problem correctly in the second run. Therefore, the main challenge is to find a strategy to communicate between the two runs using the limited output you can use. </p><p>Time limit and memory limit are <span class="tex-font-style-bf">not</span> shared between the runs. For example, in this problem, since the time limit is $$$2$$$ seconds, you will only receive a verdict of <span class="tex-font-style-tt">Time Limit Exceeded</span> if either run exceeds $$$2$$$ seconds, but it is perfectly okay if both runs of your program last $$$1.5$$$ seconds.</p><p>In this problem, your task is to find a strategy to encode and decode an array $$$a$$$ of size $$$n$$$.</p><p>The first run is the <span class="tex-font-style-bf">encoding</span> phase. You are given $$$n$$$ and the elements of $$$a$$$ by the jury. Your task is to encode this array into a string $$$s$$$ that contains only <span class="tex-font-style-bf">letters of the lowercase English alphabet</span>, and pass $$$s$$$ back to the jury. Then, your program will terminate, and all variables stored in memory will be lost.</p><p>The second run is the <span class="tex-font-style-bf">decoding</span> phase. You are given the string $$$s$$$ (the same string you passed to the jury during the first run) by the jury. Your task is to decode $$$s$$$ and determine $$$n$$$ and the elements of array $$$a$$$ that were originally given by the jury. In other words, you must reverse your encoding algorithm from the first run.</p><p><span><span class="tex-font-style-bf">First Run</span></span></p><p>Your code will be run exactly two times on each test. On the first run, you will perform encoding.</p><p><span class="tex-font-style-bf">Input</span></p><p>The first line of the input contains the string <span class="tex-font-style-tt">first</span>. The purpose of this is so your program recognizes that this is its first run, and it should act as the encoding algorithm.</p><p>The second line contains exactly an integer $$$n$$$ ($$$1 \le n \le 10^4$$$) â€” the length of $$$a$$$.</p><p>The third line contains $$$n$$$ space-separated integers $$$a_1, a_2, \ldots, a_n$$$ ($$$1 \leq a_i \leq 26$$$).</p><p><span class="tex-font-style-bf">Output</span></p><p>When ready to send $$$s$$$, you can do so by printing the following:</p><ul> <li> $$$s$$$, the encoded string ($$$1 \leq |s| \leq 10^5$$$, where $$$|s|$$$ denotes the length of $$$s$$$). This string must contain only lowercase letters of the English alphabet and should be printed without spaces. </li></ul><p>After this, your program should terminate. Note that all variables stored in memory will be lost.</p><p><span><span class="tex-font-style-bf">Second Run</span></span></p><p>On the second run, you will perform decoding. </p><p><span class="tex-font-style-bf">Input</span></p><p>The first line of the input contains the string <span class="tex-font-style-tt">second</span>. The purpose of this is so your program recognizes that this is its second run, and it should act as the decoding algorithm.</p><p>The second line of input contains string $$$s$$$, the variable that you passed to the jury during the first run.</p><p><span class="tex-font-style-bf">Output</span></p><p>When ready to send the original values of $$$n$$$ and the array $$$a$$$, you can do so by printing the following:</p><ul> <li> $$$n\, a_1\, a_2\, \ldots\, a_n$$$, the length of the original array, and the array itself. </li></ul></div><div class="sample-tests"><div class="section-title">Examples</div><div class="sample-test"><div class="input"><div class="title">Input<div title="Copy" data-clipboard-target="#id0010133963186917982" id="id002916989932202244" class="input-output-copier">Copy</div></div><pre id="id0010133963186917982">first
